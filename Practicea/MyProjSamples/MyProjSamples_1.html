Pseudo-classes and Pseudo-elements

Before I continue with ::before and ::after I should make a clear distinction between : and :: and what are pseudo-classes and pseudo-elements.

As one nice guy named scronide nicely explained on SO:

============= Quote

Pseudo-classes (:) allow you to style the different states of an element e.g. when you hover over it, when it is disabled, when it is the first child of its parent, etc.

Pseudo-elements (::) allow you to style different parts of an element e.g. the first line, the first letter, inserting content before or after, etc.

Originally they all used a single colon, but CSS3 introduced the double colon to separate the two.

============= End Quote

Other nice guys have added that for the moment browsers support both single and double colons, for backward compatibility. But we know that it is better to be compliant to the rules.

So, ::before and ::after are pseudo-elements. Very important is that the stuff is being inserted NOT before or after the element using it but INSIDE THE ELEMENT, before the content of the element and after the content of the element.

Another very important moment - THEY DO NOT WORK on images. Images according to some deep cosmic theories belong to the so called Replaced elements. Very deep stuff so that is why I keep a reference to the MDN’s source in that matter and have it available for future reading - Replaced elements .

And they (before and after) are inline by default but can be styled as block or inline-block.

CSS Variables

Beyond the basics (lets not bore both of us with that) important is the scope. They can be with global scope, accessed/used through the entire document or local variables can be used only inside the selector where they are is declared.

Global variables are declared inside the :root selector. To create a variable with local scope, declare it inside the selector that is going to use it. For ex. It can be declared inside styles for a container with a .wrapper class. So all elements within that container will be able to access it.

They can be used with a fallback value. For ex. color: var(--main-title-color, #14344b). Even with two, comma separated.

Their values can be a color, numeric value with pixels, rem-s, etc. Their values also can use another variable.

--main-bg-color: #2c1919;
--background-gradient: linear-gradient(to top, var(--main-bg-color), #222);

They can be used within calc() function. For ex. --main-content-lenght: 800px;
max-width: calc(var(--main-content-lenght) / 2);

And they can be used in bulk to change a whole theme for example. They can be switched in bulk inside a themes.css where it may contain media query for different themes, like dark or lite or inside javascript objects and change themes with javascript switching code.

Browser’s default styles, css resets and normalizers

Well, we know about the ongoing chase between browser’s creators and developers. A lot of NOT LOGICAL stuff has been created during the browsers’ creation and has to be maintained for backward compatibility.

So developers have responded in variety of ways. One approach was by using css reset. Which is scrapping browsers’ defaults.

The most basic one

* { margin: 0; padding: 0; box-sizing: border-box; }

Some experts would include border:0; font-size: 100%; font: inherit; vertical-align: baseline;

According to the experts this is the minimum to ensure same look across browsers. A lot more can be added of course. Here is one complex reset from Eric Meyer well regarded in the field - both the guy and the code.

Very important ingredient is box-sizing. It ensures that when calculating size, the padding and border will be included. As for the other values for this property - content-box does not include padding and border in the calculations. inherit speaks for itself, initial sets this property to its default value. Which is content-box.

Normalizers is one other, opposite to resets approach. Which claims - let’s have browsers’ defaults but let’s rebuild them so that hey are all the same across browsers.

Normalize.css is most popular. There was one very promising project CSS Remedy and a lot of bright minds in the field were involved. Have not checked it though for about, like half year.

Positioning, relative and absolute

By far most used properties for creating infinite number of creative, beautiful and sophisticated ideas in web design.

But first let me define a term the four directions which would mean top: x; right: x; bottom: x; left: x;. Where x can be any number, both positive or negative.

Relative - elements’ space is reserved within the flow. If we do not specify any of the the four directions the element will stay just where it is by default. If we specify any of the the four directions the element will move accordingly with respect to its default position within the flow.

Absolute - elements’ space is NOT reserved within the flow. If we do not specify any of the the four directions the element will stay just where it is by default, although other element will stuck underneath as if it is not there. If we specify any of the the four directions the element will move accordingly with respect a first met positioned parent or if there is no such, it will propagate up to to the body.

Absolute positioned element can be stretched horizontally and / or vertically with specifically chosen values for the the four directions. That is, if it does not have its own specified width and / or height.

Fixed positioning

Fixed positioned element behaves like an absolute positioned element as far as flow of the content is concerned (no space reserved). Once set with any of the the four directions it sticks relative to the viewport, or the browser window.

Just like an аbsolute positioned element, fixedcan be stretched horizontally and / or vertically with specifically chosen values for the the four directions. That is, if it does not have its own specified width and / or height.

Sticky positioning

Sticky positioned element behaves like a static or relative positioned element as far as flow of the content is concerned (has space reserved). It will move with the scroll until the scroll reaches a certain point set with any of the the four directions It sticks then relative to the viewport, or the browser window.

There can be many sticky elements on a page each pushing aside the PREVIOUS while scrolling. That is possible because:

Sticky-positioned elements are only sticky within their parent element.

Setting overflow of any of the parent elements to hidden, scroll or auto may prevent sticky from working.

Stacking Order and Z-Index

It would be a lot easier to explain with an example so I will make one video soon but until then I offer you one from one of my favorite tutors - The Net Ninja CSS Positioning Tutorial #11 - Z - Index & Stacking Order .

Turns out that the content from the begging of the HTML code has smallest stacking order and in increases as it goes further down. So if we want to place something with fixed position on the top and if we place it first within the HTML code it will go under the rest of the content. The rest of the content will have higher stacking order, i.e. z-index, kind of.

So we have two options to put something fixed on the top - either to put it at the end of the HTML code or to give it higher z-index. This use of of z-index along NOT POSITIONED elements (well not explicitly positioned) is mostly overlooked in tutorials and development cases.

The other use of z-index is when we have positioned elements vertically stacked on the top of each other but we need to correct their vertical order.

2D transforms

Transforms can transform basically three things. Shape, size, position.

The four main properties that are available:

skew(x-angle,y-angle) or skewX(angle) / skewY(angle). The way I have crunched it is - the X part is pushing toward X direction (or -X), but keeping the top and bottom sides parallel to the horizontal (X). The Y angle “lifts up” those sides (or pushes down) so they are no more parallel to the horizontal.

rotate(angle)Well, too obvious and easy ….

scale(x,y) or scaleX(x) / scaleY(y) lets us change the size or the element by multiplying its sizes to the corresponding constant given as function parameters.

translate(x,y) or translateX(n) / translateY(n)Moves the element along the corresponding axis / axes by the given parameter.

Flexbox

Well, too long and complex topic so extracting the essence. Two most important - the direction of the flow - main axis and the cross axis. Elements follow the main axis. Changing of the flow direction - horizontal / vertical come from flex-flow: row / column ….; (basically).

Some properties can be applied to the main axis and some to the cross axis. Both of those apply toward the elements as a group and change their alignment inside the flex container.

Most used for flow axis justify-content: center / flex-start / space-between /…. ;. Very similar to that but for the cross axis is align-items: center / flex-start / stretch / ….. ; .

Property like flex: flex-grow flex-shrink width(flex-basis) apply toward ONE element or group of elements, depending on the styling. Same for align-self: center / flex-start / stretch / ….. ;

Many times flex-box is used to just center items in the flex container, vertically most times as this is the easiest way nowadays.

To start laying out items in bulk inside the flex container we need to set flex-wrap: wrap;.

Flex box also allows an easy change of the order of elements in the flex container regardless of their code order with order: x. Where x can be any number, both positive or negative.

While not as powerful as the Grid, Flexbox still allows an easier way (easier than floats) of building responsive pages.
